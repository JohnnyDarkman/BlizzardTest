viewmodels/auth.js

This viewmodel manages bindings and syncs with the "services/auth.js" service. It can also be used as a parent view model to provide auth bindings and information to other nested view models.

The auth view model provides two command handlers "login" and "logout" which operate the underlying auth service to manage the user's authentication state.

The view model keeps in sync with the service using the listen feature provided by the view object which will unsubscribe from the broadcast channel when the view is destroyed.
